## Lessons learned

### L68 (NEW, candidate) — Always use `--env-file .env.docker` for compose operations

**Pattern**: My initial cutover command `docker compose up -d --force-recreate titan-portfolio` failed because the compose stack relies on env vars from `.env.docker` (TWS_USERID, IBKR_ACCOUNT_ID, UNIVERSE, etc.). Without `--env-file`, the gateway received blank credentials and hung at IBKR login.

**Rule**: ALWAYS use `docker compose --env-file .env.docker <command>` for any operation that recreates / starts containers. The compose header file documents this convention; my omission was a one-shot oversight.

**Detection**: Compose emitted a `level=warning msg="The 'TWS_USERID' variable is not set"` line — but I missed it in the cascade of "Recreate" output. Better: pre-flight a `grep` for required env vars before issuing the compose command, or write a wrapper script (e.g., `scripts/restart_portfolio.sh`) that bakes in the `--env-file` flag.
